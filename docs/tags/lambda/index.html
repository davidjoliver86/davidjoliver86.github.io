Alright.. the home stretch. At the end of the last part, we mentioned that we need a stateful data store for our Lambda function, otherwise it would keep wanting to send alerts every minute that there’s no stock, and that’s not very useful. The stateful data store we’re going to use is S3. Of course, you don’t have to use S3 - you could spin up a DynamoDB or RDS instance, and there’s the relatively new EFS on Lambda which I haven’t personally tested, but for our use case - storing an arbitrary file - S3 is my pick.
